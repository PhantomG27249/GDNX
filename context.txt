Here's the complete GDN3 feature catalog, organized by subsystem:

GDN3 (KR-MIMO-GDN2) — Full Feature List
1. State Representation
#	Feature	Description
1.1	Kronecker durable state	S = Σᵣ Aᵣ⊗Bᵣ — O(R·d) storage instead of O(d²); never materializes A⊗B
1.2	Residual exact-write buffer	UVᵀ — preserves most recent P rank-one writes exactly before compaction
1.3	Rearrangement SVD compaction	Periodic compression of UVᵀ back into Kronecker factors with exact discarded-energy monitoring
1.4	Factorized Kronecker decay	γ = γₐ ⊗ γᵦ — decay applied to A and B factors separately, maintaining Kronecker closure
2. MIMO Multi-Lane Memory
#	Feature	Description
2.1	M independent memory lanes	Default 4: local_syntax, entity_variable, reasoning_workspace, long_context_retrieval
2.2	Softmax lane router	aₜ = softmax(g(xₜ)) ∈ ℝᴹ — per-token learned routing across lanes
2.3	Lane balance regularization	L_balance = Σ(a̅ₘ - 1/M)² — prevents router collapse to single lane
2.4	Kronecker lane mixing (optional)	Z' = W_lane Z W_featᵀ — cross-lane feature transform at O(M²d + Md²) via Kronecker factorization
2.5	Per-lane independent state	Each lane maintains its own A, B, U, V, and residual pointer
3. Positional Encoding
#	Feature	Description
3.1	Partial lane-specific RoPE	RoPE applied only to first rope_fraction × d_k dimensions per lane
3.2	Per-lane RoPE fraction	Configurable: 50% (syntax), 25% (entity), 50% (reasoning), 50% (retrieval)
3.3	Per-lane RoPE frequency scaling	scaled_t = t × rope_scale — e.g., 0.2 for long-context lane (low-frequency extrapolation)
3.4	RoPE for hybrid attention	Full RoPE in sliding window attention blocks (when hybrid mode enabled)
4. Decay / Forgetting
#	Feature	Description
4.1	GDN2 channel-wise decay	Per-key-channel exponential decay α ∈ (0,1]^{dₖ}
4.2	Braided multi-timescale decay (optional)	Mixture of M_braid=4 log-spaced exponential rates approximating power-law (1+d)^(-γ)
4.3	Learned braiding router	Per-token weights blend fast/local vs slow/global decay timescales
4.4	Kronecker-closed factorization	Braided decay factorized into γₐ ⊗ γᵦ via geometric mean over factor axes
5. Write Coefficient / Discretization
#	Feature	Description
5.1	Exact exponential coefficient	α = (1-e⁻ᶜ)/c — exact ODE solution for rank-one vector field
5.2	Numerically stable limit	Taylor series δ(1 - z/2 + z²/6) for `
5.3	Alternative modes	Euler, Heun (explicit trapezoid), implicit trapezoid (Tustin)
5.4	Configurable delta	Step size δ (default 1.0)
6. Gating
#	Feature	Description
6.1	Channel-wise erase gate bₜ	Key-axis selective read-removal, [0,1]^{dₖ} per lane
6.2	Channel-wise write gate wₜ	Value-axis selective commit, [0,1]^{dᵥ} per lane
6.3	Per-lane gate projections	Independent b_proj and w_proj per lane
6.4	Sigmoid squashing	Gates constrained to [0,1] via torch.sigmoid
7. Channel Generation
#	Feature	Description
7.1	Standard dense projections	Baseline q, k, v projections (always present)
7.2	Hopf-inspired coproduct routing (optional)	Kronecker-factored channel generation: z = Σ s_c · vec(a_c(x) ⊗ b_c(x)) for bilinear feature binding
7.3	Coproduct blend mixing	Learnable coprod_u_mix, coprod_v_mix parameters (default 0.5) blend dense + coproduct
7.4	Coproduct normalization	RMSNorm or L2 normalization on coproduct output
8. Kernel / Computational
#	Feature	Description
8.1	Kronecker read identity	(A⊗B)·x = vec(BX Aᵀ) — avoids materialization via two bmm calls (K-FAC form)
8.2	Post-update output shortcut	y = Sq + α·r(kᵀq) — computes output without re-reading updated state
8.3	L2 normalization	q and k normalized for numerical stability
8.4	Mixed precision	BF16 for factors/residuals, FP32 for accumulators and compaction SVD
8.5	K-FAC bmm forward path	torch.bmm-based Kronecker read — 593 tok/s at d=128
8.6	Circular residual buffer	Fixed-size P slots with wrapping pointer
9. Model Integration
#	Feature	Description
9.1	GDN3Block	Transformer block: GDN3 mixer → RMSNorm → MLP → RMSNorm
9.2	GDN3HybridBlock	Recurrent + sliding window attention (configurable window size)
9.3	GDN3Model	Full LM stack: token embeddings → stacked blocks → LM head
9.4	Warm-start from GDN2	Lane 1 initialized from GDN2 weights; other lanes as small perturbations
9.5	Router initialization	a₁ ≈ 0.85, aₘ>₁ ≈ 0.15/(M-1) with entropy regularization
10. Diagnostics & Regularization
#	Feature	Description
10.1	Compaction error monitoring	ε² = Σ_{r>R} σᵣ² logged per layer/head/lane
10.2	Residual saturation tracking	How often pointer hits P
10.3	Alpha statistics	Distribution of c = kᵀ(b⊙k) and resulting α
10.4	Lane specialization metrics	Cosine similarity between lane outputs
10.5	Lane balance loss	Weak regularizer to prevent collapse
11. Testing & Verification
#	Feature	Description
11.1	12 dense equivalence tests	KronRead identity, residual exactness, decay closure, exact alpha, output shortcut, compaction error, full lane step, alpha modes, braided decay, partial RoPE, numerical gradient, bandwidth analysis
11.2	Throughput benchmark	Token-per-second across configs
11.3	Bandwidth benchmark	State traffic verification vs dense
11.4	Needle retrieval benchmark	Planted-needle long-context retrieval
11.5	Multi-key tracking benchmark	Variable/entity tracking across context
